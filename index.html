Master's Thesis  ·  Adversarial Defense 2024 – 2025

BFDM — BEV Feature Denoising Module

Plug-and-play adversarial defense for multi-sensor 3D object detection in autonomous driving

Problem

BEVFusion (camera + LiDAR fusion) drops from NDS 0.52 to 0.32 and mAP 0.49 to 0.19 under AutoPGD-20 white-box attack. Existing defenses require backbone retraining — expensive and impractical once a model is deployed.

Approach

Two modules inserted after BEV feature extraction, requiring no backbone modification: PNLM — point-wise neighbor local matching via dynamic routing to suppress locally corrupted regions; AMHSA — adversarial multi-head self-attention with dilated convolutions to reconstruct semantically destroyed BEV structures.

Constraints & Trade-offs

Adds 0.17M parameters and <8ms inference overhead on nuScenes. Defense evaluated under white-box adaptive attack (AutoPGD-20). Black-box and physical-world attack scenarios not fully validated — an honest limitation.

Setup

nuScenes v1.0 dataset. BEVFusion backbone (frozen). Evaluates NDS and mAP under both clean and adversarial conditions. White-box adaptive attacker has full model access.

Vehicle Accident Prediction System

Early accident prediction from dashcam footage — 2023 IEEE ICME AVA Challenge winner (1/12)

Problem

Given dashcam footage, predict whether an accident will occur within the next 2 seconds. Dataset: 733 videos at 60 fps, mixed resolutions (640×480 to 1920×1080), with severe XML annotation inconsistencies requiring a full manual correction pass before training.

Approach

YOLOv7-E6E as the base detector. Added two auxiliary signals derived from physical motion laws: (1) per-object bounding-box area growth rate across frames, (2) proximity relationships between bounding boxes. Neither required retraining the detector — they were post-processing heuristics layered on top.

Data Work

XML annotations across 733 videos had inconsistent timestamp offsets and naming-order bugs. Built a custom script to detect and fix ordering issues, then reviewed ambiguous cases manually. The cleaning pass consumed roughly 30% of total project time.

Honest Reflection

The winning margin came primarily from the physical-rule heuristics, not from a superior detector. A stronger baseline model would have had limited marginal gain. The real leverage was in the post-processing logic — which is a useful reminder about where problems actually live.

Fisheye Object Detection — PAIR-LITEON

Knowledge distillation for embedded fisheye detection with annotation-scarce data

Problem

Fisheye camera data is annotation-scarce and models must run on MemoryX embedded AI hardware with strict latency and size budgets. Standard perspective-trained detectors don't generalize well to fisheye distortion.

Approach

YOLOv7 backbone + PRB-FPN neck with redundant layer pruning (52% model size reduction). Semi-pseudo-label distillation: dynamic quality-score filtering generates synthetic labels from unlabeled fisheye pool to train a lighter student model.

Embedded Deployment

Quantized and deployed on MemoryX platform. The hardware constraint forced most architectural decisions — you can't iterate on a target device as quickly as you can in a GPU cluster, so wrong bets are expensive.

Most Sensitive Variable

The pseudo-label quality threshold. Too permissive → noisy training signal and accuracy collapse. Too strict → insufficient pseudo-labels, model starves. The right threshold varied significantly across fisheye scene types — no stable heuristic found.

Undergraduate Capstone  ·  AIoT System 2022

Smart Goose Farming Management System

AIoT system for automated behavior monitoring and anomaly detection in livestock farming

Architecture

Raspberry Pi 4 running YOLOv5s (detection) + DeepSORT (multi-object tracking). MQTT for sensor telemetry. RESTful API for the management interface. OpenCV for camera-stream distortion correction (color-difference calibration).

Anomaly Detection

Time-series residual model with dynamic threshold — adapts to baseline behavior patterns established from the first N frames. Avoids hard-coded rules for behavioral abnormalities, which vary by flock size and season.

Constraint: Edge Compute

Raspberry Pi 4 compute budget forced YOLOv5s over more accurate variants. False-positive rate on small flocks (<10 geese) was a known limitation — the anomaly model was calibrated for 30+ animals.

Outcomes

3rd nationally (3/969) at 2022 ICT Innovative Services Awards. Best Practical Work at 2021 national competition. Published at CVGIP 2022. The system runs on commodity hardware with no cloud dependency.
